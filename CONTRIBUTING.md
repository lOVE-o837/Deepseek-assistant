# 贡献指南

感谢你对 DeepSeek 助手 的关注！欢迎通过以下方式参与贡献：

## 报告 Bug
如果你发现了任何问题，请在 [Issues](https://github.com/lOVE-o837/DeepSeek-Assistant/issues) 页面提交，并尽可能包含：
- 问题现象的详细描述
- 复现步骤（从哪一步开始，做了什么操作）
- 你的环境信息：Edge 浏览器版本、插件版本、操作系统
- 相关截图或控制台错误信息

## 功能建议
如果你有新功能的想法，欢迎在 [Issues](https://github.com/lOVE-o837/DeepSeek-Assistant/issues) 中提出，标题前加上 `[功能建议]`。请描述你希望实现的功能以及它能解决什么问题。

## Pull Request 流程
如果你想直接贡献代码，请按以下步骤操作：
1. **Fork 本仓库**：点击页面右上角的 Fork 按钮，将项目复制到你自己的账户下。
2. **克隆到本地**：`git clone https://github.com/你的用户名/DeepSeek-Assistant.git`
3. **创建功能分支**：`git checkout -b feature/你的功能名称`
4. **提交修改**：`git commit -m '添加了某某功能'`
5. **推送到远程**：`git push origin feature/你的功能名称`
6. **创建 Pull Request**：回到原仓库，点击 "New Pull Request"，选择你的分支并提交。

## 代码风格
- 使用 ES6+ 语法
- 遵循 Manifest V3 规范
- 保持代码简洁，添加必要的注释

## 行为准则
参与本项目即表示你同意遵守我们的 [行为准则](./CODE_OF_CONDUCT.md)。请保持友善，尊重所有贡献者。
